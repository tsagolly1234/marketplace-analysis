# 🛒 Анализ маркетплейса

## О проекте
Стартап, занимающейся продажей товаров из Бразилии.
Выручка компании стоит на месте несколько месяцев нужно понять почему и предложить решение.

## Задачи
1. Когортный анализ месячного retention на этапе оформления заказа
2. Оценка наличия product/market fit
3. Определение 5 ключевых метрик для максимизации прибыли
4. Приоритизация гипотез с помощью фреймворка ICE
5. Формулировка целевых, proxy и guardrail метрик для выбранной гипотезы
6. Итог с выводами и рекомендациями

## Ключевые выводы
- PMF отсутствует: пользователи делают одну покупку и не возвращаются, кривая retention не выходит на плато
- Наивысший приоритет по ICE (score: 192) — исправление бага в системе процессинга заказов
- Масштабирование и выход на новые рынки нецелесообразны до достижения retention ≥ 5%

## Стек
`Python` `Pandas` `NumPy` `Matplotlib` `Seaborn`  
Когортный анализ · Продуктовые метрики · ICE-фреймворк

## Данные
Датасет Olist (бразильский маркетплейс) 3 таблицы:
- `olist_customers_dataset.csv` — пользователи
- `olist_orders_dataset.csv` — заказы
- `olist_order_items_dataset.csv` — товарные позиции
